Telegram Integration Documentation

Complete guide to configuring webhooks, alerts, and remote access generation.

1. Introduction

Guard Helper features a robust, secure Telegram Integration module. This allows system administrators to monitor security events, receive critical alerts, and programmatically generate token-based invitation links directly inside a restricted Telegram Chat or Channel.

2. Telegram Bot Commands

Once configured and registered, the Telegram Bot listens for text commands in the authorized chat room. The following commands are supported:

  • /start or /help
    Returns the welcome screen detailing all available bot commands and their usage syntax.
  • /generate <email> <platform> [limit]
    Generates a new token-based Access Grant. If the platform exists and the email is valid, the bot sends back a secure, clickable access link. Default limit is 20 if omitted.
  • /token <email> <platform> [limit]
    Alias for the /generate command.

Example command usage:

/generate user@domain.com Steam 15

Expected response:

✅ Access Token Generated

📧 Email: user@domain.com
🎮 Platform: Steam
🔢 Usage Limit: 15 times

🔗 Access Link:
https://yoursite.com/grab-code?token=gh_tok_abcd12345...

3. Webhook Integration

Telegram Webhook allows your application to receive message payloads from Telegram instantly. When a user sends a command to your Bot, Telegram dispatches an HTTP POST request to your webhook endpoint.

POST Endpoint Route

{APP_URL}/api/webhook/telegram/message

Security & Access Control

To prevent unauthorized users from querying codes or generating invitations, the system performs strict validation checks on incoming webhook payloads:

  1. Chat ID Authorization: The webhook compares the sender's chat.id with the telegram_chat_id saved in settings. If they do not match, the request is rejected with a warning log, and the sender receives an "Access Denied" reply.
  2. Token Configuration: Webhook messages are ignored if the system's telegram_bot_token is not properly set.

4. Managing Webhook Registration

In the System Settings panel, you can register or unregister your webhook endpoint directly with Telegram using the single-button toggler.

  • Register Webhook: Sends a registration call to Telegram using your configured telegram_bot_token and your site's APP_URL.
  • Unregister Webhook: Removes the callback URL registration from Telegram, turning off active bot commands.

⚠️ Localhost Environments Blocked

Telegram's setWebhook API strictly requires a publicly accessible, secure HTTPS URL. If your server is running on localhost or 127.0.0.1, webhook registration is automatically disabled.

💡 Tip: Local Testing with Ngrok

To test Telegram commands locally, start a secure tunnel using ngrok:
ngrok http 8000
Then, copy the generated HTTPS URL (e.g., https://abcde.ngrok-free.app), update your APP_URL in your Laravel .env file, and save settings. The webhook toggler will now allow activation.

5. Notification & Alert System

When the Telegram module is active (and telegram_enabled is toggled on in settings), Guard Helper sends real-time alert messages to the authorized chat room for critical background events:

  • IMAP Connection Failures: Alerts you immediately if the system cannot connect to a custom IMAP mail server.
  • Gmail/Outlook OAuth Expirations: Warns you if an authentication token is revoked or expired.
  • Security Interception Failures: Notifies you if an unauthorized user attempts to grab a code for a platform they are not assigned to.
